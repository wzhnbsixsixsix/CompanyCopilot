# 知识库问答技能

## 技能概述
本技能为智能助手提供基于用户上传文档的知识检索和问答能力。当用户使用 `/kg` 命令时，系统将从知识库中检索相关信息，并基于检索结果生成准确、有据可查的回答。

## 使用场景
- 用户需要基于已上传文档查询特定信息
- 用户希望从多个文档中综合提取相关内容
- 用户需要对文档内容进行问答式交互

## 支持的文档类型
- **文本文档**: PDF, TXT, Markdown, DOCX
- **表格数据**: XLSX, CSV
- **演示文稿**: PPTX
- **图片文档**: JPG, PNG, JPEG, WEBP (通过 VL 模型提取内容)

## 工作流程

### 1. 理解用户问题
- 分析用户的查询意图
- 识别关键词和查询要素
- 必要时对问题进行改写以优化检索效果

### 2. 知识检索
使用 `retrieve_knowledge` 工具从知识库中检索相关文档块：
- 设置合适的 `limit` 参数（推荐 3-5）
- 设置 `score_threshold` 过滤低相关性结果（推荐 0.3-0.5）
- 如果初次检索结果不佳，尝试改写查询关键词

### 3. 信息整合
- 从检索结果中提取与问题相关的信息
- 综合多个文档块的内容
- 注意信息的一致性和完整性

### 4. 生成回答
- 基于检索到的信息生成回答
- 明确标注信息来源
- 如果检索结果不足以回答问题，如实告知用户

## 工具使用规范

### retrieve_knowledge 工具
```
参数说明：
- query: 检索查询文本（必需）
- limit: 返回结果数量上限（默认 5，范围 1-20）
- score_threshold: 相关性分数阈值（默认 0.3，范围 0-1）
```

### 检索策略
1. **初次检索**: 使用用户原始问题作为查询
2. **优化检索**: 如果结果不理想，尝试：
   - 提取核心关键词重新查询
   - 使用同义词或相关术语
   - 拆分复杂问题为多个简单查询

## 回答规范

### 信息引用
- 每个关键信息点标注来源文件
- 使用格式：`（来源: 文件名）`
- 多个来源时列出所有相关文件

### 不确定性处理
- 如果检索到的信息不足，明确说明
- 对于模糊或矛盾的信息，提供多个可能的解释
- 避免编造或推测未经文档支持的信息

### 回答结构
1. **直接回答**: 简洁明了地回答用户问题
2. **详细解释**: 提供相关背景和详细信息
3. **信息来源**: 列出引用的文档来源
4. **补充说明**: 如有必要，提供相关建议或注意事项

## 示例对话

### 示例 1: 基本问答
```
用户: /kg 公司的年度营收是多少？

助手: 根据知识库中的文档，公司 2025 年度营收为 1.5 亿元人民币，
同比增长 25%。（来源: 年度财务报告.pdf）

主要营收来源：
- 产品销售: 1.2 亿元 (80%)
- 服务收入: 0.3 亿元 (20%)
（来源: 年度财务报告.pdf, 业务分析.xlsx）
```

### 示例 2: 无法回答
```
用户: /kg 竞争对手的市场份额是多少？

助手: 抱歉，在当前知识库的文档中没有找到关于竞争对手市场份额的
相关信息。您可能需要上传包含此信息的文档，或者通过其他方式获取
这些数据。

如果您有相关文档，可以通过文件上传功能添加到知识库中。
```

## 注意事项
- 始终基于检索到的文档内容回答，不要凭空编造
- 保持客观中立，如实反映文档内容
- 对于敏感或机密信息，提醒用户注意信息安全
- 如果文档内容存在错误或过时，建议用户核实更新
